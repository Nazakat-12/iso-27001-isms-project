## Statement of Applicability (SoA)

The following table documents the selected controls from ISO/IEC 27001:2022 Annex A, justified by the risk assessment (see Risk Register) and organizational requirements.

| Control ID | Control Name | Justification for Inclusion | Implementation Status | Excluded? |
| :--- | :--- | :--- | :--- | :--- |
| **A.5.7** | Threat intelligence | Required to proactively inform the Incident Response Procedure (IRP). | Implemented | No |
| **A.6.5** | Contact with authorities | Necessary for legal compliance and handling major incidents. | Planned | No |
| **A.8.10** | Data masking | Required to comply with GDPR/CCPA for development and testing environments. | Implemented | No |
| **A.8.17** | Development testing | Essential to ensure code quality and security before deployment. | Implemented | No |
| **A.8.27** | Secure disposal of media| Not applicable, as all development is cloud-native; no physical media is stored/recycled. | N/A | Yes |

> **Note:** The final column provides the cross-reference to the policy or procedure where the control is implemented (e.g., A.5.7 is implemented in the Incident Response Procedure).
